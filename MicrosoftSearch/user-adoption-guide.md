# <a name="user-adoption-guide"></a>Guía de adopción del usuario

Esta información está dirigida a administradores de Microsoft Search, líderes de administración de cambios y propietarios de empresas.
  
Para implantar y garantizar la adopción correcta de Microsoft Search en toda la organización, se recomienda este proceso de cuatro fases.
  
## <a name="stage-1-stakeholders"></a>Fase 1: Stakeholders

Para garantizar una implementación sin problemas y aumentar la adopción por parte del usuario de Microsoft Search, identifique a las personas que pueden ayudar a admitir e impulsar el proceso.
  
En la tabla se muestra la jerarquía del equipo y lo que cada persona contribuye. 
  
 **Roles y responsabilidades del equipo de adopción de usuarios**
  
|**Rol**|**Responsabilidades**|**Departamento**|
|:-----|:-----|:-----|
|Patrocinador ejecutivo  <br/> |Comprenda la visión de alto nivel y el valor de Microsoft Search para su organización.  <br/> Proporcionar soporte de liderazgo al equipo de IT/business.  <br/> Asignar recursos.  <br/> Comunicarse directamente con los empleados.  <br/> |Liderazgo ejecutivo  <br/> |
|Propietario de empresa  <br/> |Identificar escenarios de producto y usuario.  <br/> Determinar las métricas de éxito de los departamentos y la organización.  <br/> |Cualquiera  <br/> |
|Equipo de administración de cambios  <br/> |Asegúrese de que el valor del producto sea fácil de comprender.  <br/> Cree un plan para el lanzamiento.  <br/> |TI  <br/> |
|Expertos:  <br/> |Evangelizar el producto a compañeros y equipos.  <br/> |Cualquiera (personal)  <br/> |
|Administrador de búsqueda  <br/> |Identificar los miembros principales del equipo de adopción de usuarios.  <br/> Administra Microsoft Search, incluida la apariencia de la experiencia y la administración continua.  <br/> Impulsar discusiones para mostrar el valor de Microsoft Search.  <br/> |TI  <br/> |

## <a name="stage-2-scenarios"></a>Fase 2: Escenarios

Para garantizar la adopción del usuario, es fundamental determinar y definir escenarios de usuario final. Estos deben incluir ejemplos reales de cómo los usuarios pueden usar Microsoft Search para encontrar la información que necesitan, responder preguntas y completar tareas con mayor rapidez. Para medir el éxito de la implementación y la adopción del usuario, también es importante determinar las métricas que usarás antes de comenzar.
  
### <a name="understand-what-your-users-need-and-want"></a>Comprender lo que los usuarios necesitan y desean
  
Cuando se trata de buscar información y obtener respuestas, es clave conocer las necesidades y los puntos de dolor del usuario. Puede usar métodos cuantitativos, como datos del uso de búsqueda de [Office 365](https://Office.com) y [SharePoint,](https://products.office.com/sharepoint/collaboration) o métodos cualitativos, como encuestas y grupos de enfoque de usuario para obtener estos datos. Esto le ayudará a determinar cuándo, dónde y cómo Microsoft Search puede proporcionar valor.
  
Para distintos roles y equipos de toda la organización, tenga en cuenta:
  
- Los puntos de dolor del usuario existentes al buscar información.
    
- Aplicaciones, herramientas, sitios y otros orígenes de información usados con frecuencia.
    
- Lo que los usuarios quisieran ver cuando se trata de buscar información.
    
- Cómo se aplican las ventajas de Microsoft Search a su organización. Para obtener más información, vea [Microsoft Search Overview](overview-microsoft-search.md).
    
Elija los escenarios de usuario que tendrán el mayor impacto positivo para los usuarios.
  
 ### <a name="identify-success-metrics"></a>Identificar métricas de éxito
  
Para ayudarle a cumplir los objetivos de implementación y mostrar un impacto positivo para su organización, es fundamental determinar las métricas clave de éxito antes de empezar. Mantenga sus objetivos simples y medibles, y evalúe el rendimiento de forma regular, para que pueda responder rápidamente si es necesario.
  
1. **Administración de cambios:** Determine cómo medirá el conocimiento de Microsoft Search y el éxito de las comunicaciones al respecto. 
    
2. **Uso:** Puede aprovechar los datos del Panel de insights, incluidos los usuarios activos diarios y mensuales (DAU y MAU), los tipos de contenido que más vistas están obteniendo y las palabras clave de búsqueda superiores para medir el uso con el tiempo. Para obtener más información, vea [Informes del panel de Microsoft Search Insights](./usage-reports.md).
    
3. **Satisfacción del usuario:** Use los datos recopilados de encuestas y grupos de enfoque de usuario para determinar la línea base. La satisfacción neta (NSAT) y los resultados de Net Promoter (NPS) también pueden proporcionar medidas útiles. Independientemente del método que use, mantenga la coherencia, para que pueda ver el impacto con el tiempo. 
    
4. **Otras medidas:** Use datos históricos o actuales para solicitudes de soporte técnico y la cantidad de tiempo invertido en la búsqueda. 
    
## <a name="stage-3-readiness"></a>Fase 3: Preparación

Invertir tiempo en preparar el lanzamiento de Microsoft Search tendrá un impacto positivo para la organización. Microsoft proporciona un <a href="https://go.microsoft.com/fwlink/?linkid=2114710">Kit de adopción</a> que contiene una plantilla de correo electrónico de anuncio, contenido visual ligero creado para compartir en los sitios de comunicación interna y una presentación de aprendizaje. También hay un Centro de recursos de <a href="https://resources.techcommunity.microsoft.com/microsoft-search/#adoption">Microsoft Search</a> donde puede encontrar otros recursos de aprendizaje y adopción diseñados para ayudar a que su implementación de Microsoft Search sea correcta.
  
### <a name="product-and-content-readiness"></a>Preparación de productos y contenido
  
Los administradores influyen directamente en la experiencia de búsqueda de los usuarios finales y determinan cómo busca Microsoft Search para la organización. Esto incluye la elección de los tipos de resultados que quiere que se muestren a los usuarios. Estos resultados pueden incluir vínculos (también conocidos como marcadores) a documentos, sitios, personas, grupos, ubicaciones, conversaciones y otros recursos del lugar de trabajo.
  
Para obtener más información, vea [Configurar Microsoft Search](setup-microsoft-search.md).
  
Los editores son expertos en la materia que ayudan a administrar marcadores y preguntas&Como dentro de sus áreas de experiencia. Los editores deben crear contenido que se alote en los escenarios de usuario que identificó anteriormente.
  
Para obtener más información, vea [Plan your content](plan-your-content.md).
  
### <a name="signing-in"></a>Iniciar sesión
  
Los usuarios de Microsoft Search deben iniciar sesión en [Bing](https://Bing.com) con una cuenta laboral o educativa. Para que el inicio de sesión sea lo más transparente posible para los usuarios:
  
- Para asegurarse de [que las configuraciones de Azure Active Directory](/azure/active-directory/) están configuradas correctamente, haga que algunos usuarios intenten iniciar sesión. Para obtener más información, vea [Test single sign-on](test-single-sign-on.md).
    
- Establece los valores predeterminados del explorador para dispositivos administrados por la empresa en Bing. Para obtener más información, vea [Set default browser](/deployedge/edge-default-browser).
    
- Implementar extensiones de búsqueda para [usuarios de Microsoft Edge](https://www.microsoft.com/windows/microsoft-edge) y Google [Chrome](https://www.google.com) a través de la directiva de grupo. Para obtener más información, vea [Microsoft Search Overview](overview-microsoft-search.md).
    
### <a name="draft-communications"></a>Borrador de comunicaciones
  
Usa los escenarios reales para contar a los usuarios acerca de Microsoft Search y cómo les ayudará a ahorrar tiempo. El equipo de administración de cambios puede ayudar a crear una estrategia de lanzamiento y preparar las garantías y las campañas para la preparación del usuario final.
  
Comience a planear comunicaciones y eventos para su organización:
  
- Coloque banners, pósteres o folletos en ubicaciones destacadas.
    
- Demostraciones de productos en persona o stands.
    
- Eventos en línea para equipos locales y remotos.
    
- Que el patrocinador ejecutivo hoste una reunión del ayuntamiento o envíe un correo electrónico de anuncio.
    
- Compartir documentación de auto help sobre el uso de Microsoft Search.
    
Todas las comunicaciones deben informar a los usuarios sobre cómo pueden iniciar sesión en [Bing](https://Bing.com) para encontrar resultados de trabajo.
  
### <a name="engage-partner-teams"></a>Participar en equipos asociados
  
Como parte integral de [Microsoft 365,](https://www.microsoft.com/microsoft-365)Microsoft Search cumple con las directrices del RGPD y del nivel C. Según la organización, es posible que sea necesario compartir información sobre Microsoft Search con los equipos de privacidad y seguridad. Para obtener más información, vea [Seguridad y privacidad en Búsqueda de Microsoft en Bing](security-for-search.md).
  
Asegúrese de que el departamento de soporte técnico conozca las características y funcionalidades de Microsoft Search, así como los requisitos de usuario para iniciar sesión, para que puedan admitir a los usuarios de forma más eficaz. Para obtener más información, vea [Learn about Microsoft Search in Microsoft 365](https://support.office.com/article/Learn-about-Microsoft-Search-in-Microsoft-365-b8bf5a2c-7515-40a9-9a6a-b8ed382c86bc) and [FAQs](faqs.md).
  
### <a name="evaluate-with-pilot-users"></a>Evaluar con usuarios piloto
  
Comience con un pequeño grupo de usuarios para evaluar el impacto de Microsoft Search y recopilar comentarios anticipados. Esto le da tiempo para cambiar el contenido, actualizar las comunicaciones y establecer expectativas para el lanzamiento general. Los usuarios avanzados de búsqueda o los primeros usuarios son excelentes candidatos para ello.
  
Para ayudar a encontrar usuarios piloto:
  
- Asocie con un patrocinador empresarial dentro de su organización.
    
- Identificar un grupo de usuarios iniciales.
    
- Identifique grupos que realicen grandes volúmenes de búsquedas empresariales.
    
- Identificar usuarios en otras ubicaciones geográficas, incluidos los internacionales.
    
Pida a los usuarios piloto que proporcionen información con las opciones de comentarios de Microsoft Search o mediante encuestas sencillas.
  
Preguntas de encuesta para hacer: 
  
- ¿Comprende cómo iniciar sesión y usar Microsoft Search?
    
- ¿Funciona como se esperaba?
    
- ¿Qué características te gustan?
    
- ¿Pudo encontrar resultados de trabajo?
    
- ¿Resultaron útiles los resultados del trabajo?
    
- ¿Las comunicaciones sobre Microsoft Search son claras y comprensibles?
    
Este es un buen momento para empezar a medir la eficacia de las métricas identificadas en la fase 2.
  
Cuando inicie la fase piloto, considere la posibilidad de compartir un vistazo de Microsoft Search con la organización más amplia.
  
## <a name="stage-4-roll-out"></a>Fase 4: Lanzamiento

Cada organización piensa de forma diferente en los lanzamientos de productos. Estos son algunos aspectos a tener en cuenta.
  
### <a name="broad-deployment"></a>Amplia implementación
  
Cuando esté listo, permita que todos los usuarios de su organización accedan a Microsoft Search o use un enfoque por fases para su implementación más lentamente. 
  
### <a name="generate-awareness"></a>Generar conciencia
  
Genere entusiasmo en su organización sobre Microsoft Search:
  
- Enliste usuarios piloto para proporcionar asistencia y experiencia punto a punto.
    
- Siga las comunicaciones y los eventos que planeó en la fase 3.
    
### <a name="ongoing-communication"></a>Comunicación continua
  
Durante las primeras semanas y meses, mantenga un ritmo de comunicaciones para fomentar la interacción con Microsoft Search. Esto mantiene a los usuarios informados y a los nuevos usuarios. Cada comunicación debe explicar por qué Microsoft Search puede ser útil para su trabajo.
  
Envíe comunicaciones en curso que resalte las características y las búsquedas para probar, así como los casos de éxito del usuario.
  
### <a name="gather-feedback-and-improve"></a>Recopilar comentarios y mejorar
  
Apoye a los usuarios y déjeles saber que le importa escucharlos y actuar sobre sus comentarios.
  
Use el panel de información para ver el rendimiento del producto y el contenido. Aproveche esta información para actualizar marcadores, preguntas&As y otras comunicaciones. Esto también le ayudará a:
  
- Mejore la calidad del contenido: revise títulos, descripciones y direcciones URL, optimice las palabras clave, quite información obsoleta y agregue nueva información.
    
2. Determine qué características se usan con más frecuencia o con poca frecuencia. 
    
3. Realice encuestas, grupos de enfoque y sesiones de comentarios para comprender lo que los usuarios piensan de Microsoft Search, aumentar la adopción e identificar áreas de mejora.
    
Educación y comentarios es un proceso iterativo. Es posible que sea necesario realizar cambios en el contenido y las comunicaciones a medida que los usuarios adopten Microsoft Search.
